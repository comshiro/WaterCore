**Minimum Required Stack (for a strong hackathon MVP)**

1. Copernicus data discovery and download
- API: Copernicus Data Space Ecosystem STAC API
- URL: https://catalogue.dataspace.copernicus.eu/stac
- Use for: finding Sentinel-1 and Sentinel-2 scenes by area/time/cloud cover
- Python package: pystac-client

2. Copernicus data processing
- API: Sentinel Hub Process API
- URL: https://services.sentinel-hub.com/api/v1/process
- Use for: requesting ready-to-use rasters (NDVI, NDWI, custom flood masks) without building your own heavy preprocessing stack
- Python package: sentinelhub

3. Climate and hydrological baselines
- API: Copernicus Climate Data Store API
- Use for: ERA5 rainfall/temperature baselines, drought anomaly context, seasonal comparisons
- Python package: cdsapi

4. Galileo/EGNOS positioning evidence
- Source: GNSS device stream (receiver or phone) using Galileo-enabled observations
- Use for: trusted geotagging of field reports, sensor locations, and flood marks
- Python packages: pyserial, pynmea2
- Optional precision path: NTRIP corrections via pygnssutils or external RTK tools

5. Geospatial analytics
- Libraries: rasterio, rioxarray, xarray, geopandas, shapely, numpy, pandas
- Use for: clipping AOI, index calculations, time aggregation, risk scoring

6. Map and alert delivery
- Backend: FastAPI
- Frontend map: Leaflet or MapLibre (or quick MVP with folium)
- Use for: risk map endpoint, alert endpoint, user-facing dashboard

**API Access Setup (what accounts/keys you need)**

1. Copernicus Data Space account
- Create account on Copernicus Data Space
- Use credentials/token for catalogue and downloads

2. Sentinel Hub account
- Create account and OAuth client
- Store client id and client secret in environment variables

3. CDS account
- Register at Copernicus Climate Data Store
- Configure CDS API key locally for cdsapi client

4. Galileo data source
- Ensure your GNSS receiver/phone provides Galileo constellation observations
- If using correction streams, register with an NTRIP provider (and optionally EGNOS-related service access if available in your region)
